## 1. 產品概述

### 1.1 產品定位
本系統是一套**線上協作撰書平台**，讓多位作者能夠共同編寫同一本書籍，支援書籍與章節的結構化管理、即時多人共筆、章節內容編輯，並在編輯介面右側提供寫作數據統計與多媒體素材管理，協助寫作團隊掌握進度與內容品質。

### 1.2 解決的問題
- 傳統文件協作（如 Word 來回傳檔）難以管理版本與多人衝突。
- 書籍寫作缺乏「以章節為單位」的結構化管理與進度可視化。
- 缺少即時的字數、進度等寫作數據回饋。
- 文字與多媒體素材分散，難以集中管理與引用。

### 1.3 目標使用者
| 角色 | 說明 | 主要訴求 |
|------|------|----------|
| 主編 / 書籍擁有者 | 建立並管理書籍，分配權限 | 結構管理、進度掌控 |
| 共同作者 | 撰寫指定章節內容 | 順暢的編輯體驗、不衝突 |
| 審閱者 | 檢視、評論、提出修改建議 | 評論、檢視差異 |
| 一般讀者（選配） | 唯讀檢視已發布內容 | 閱讀、檢索 |

## 2. 產品目標與成功指標

### 2.1 產品目標
1. 提供穩定、低衝突的多人即時協作編輯體驗。
2. 以「書籍 → 章節 → 內容」三層結構清楚管理寫作專案。
3. 透過右側資訊面板讓作者即時掌握寫作數據與素材。

### 2.2 成功指標（KPI）
| 指標 | 目標值 |
|------|--------|
| 多人同時編輯衝突率 | < 1% |
| 編輯內容自動儲存延遲 | < 2 秒 |
| 章節載入時間 | < 1.5 秒 |
| 月活躍協作書籍數 | 上線 3 個月達 500 本 |
| 使用者次月留存率 | > 40% |

### 3.1 功能總覽
| 模組 | 功能 | 優先級 |
|------|------|--------|
| 書籍管理 | 建立 / 編輯 / 刪除 / 列表書籍 | P0 |
| 成員與權限 | 邀請成員、角色分配、權限控制 | P0 |
| 章節管理 | 新增 / 重新命名 / 排序 / 刪除章節 | P0 |
| 內容編輯 | 富文本編輯、多人共筆、自動儲存 | P0 |

| 右側多媒體面板 | 導入圖片 / 影片 / 音訊 / 檔案 | P1 |

### 3.2 書籍管理（P0）

**需求描述**：使用者可建立並管理多本書籍，每本書為獨立的協作專案。

**功能細項**
- 建立書籍：輸入書名、簡介、封面、分類 / 標籤。
- 書籍列表：以卡片 / 列表顯示，可搜尋、排序（最近編輯、建立時間、書名）。
- 編輯書籍：修改書名、簡介、封面等中繼資料（metadata）。
- 刪除書籍：軟刪除（移入回收桶），可還原；僅擁有者可永久刪除。
- 書籍狀態：草稿（Draft）/ 進行中（Writing）/ 已完成（Completed）/ 已封存（Archived）。

**驗收標準**
- 使用者可在 3 秒內完成建立一本新書並進入編輯畫面。
- 刪除後 30 天內可從回收桶還原。

---

### 3.3 成員與權限管理（P0）

**需求描述**：支援多人協作，依角色控管操作權限。

**角色與權限矩陣**
| 操作 | 擁有者 Owner | 編輯者 Editor | 審閱者 Reviewer | 讀者 Viewer |
|------|:---:|:---:|:---:|:---:|
| 刪除書籍 | ✅ | ❌ | ❌ | ❌ |
| 管理成員 / 權限 | ✅ | ❌ | ❌ | ❌ |
| 新增 / 刪除章節 | ✅ | ✅ | ❌ | ❌ |
| 重新命名章節 | ✅ | ✅ | ❌ | ❌ |
| 編輯內容 | ✅ | ✅ | ❌ | ❌ |
| 評論 | ✅ | ✅ | ✅ | ❌ |
| 檢視內容 | ✅ | ✅ | ✅ | ✅ |
| 導入多媒體 | ✅ | ✅ | ❌ | ❌ |

**功能細項**
- 透過 Email 或連結邀請成員加入書籍。
- 擁有者可隨時調整成員角色或移除成員。
- 顯示目前線上協作的成員（在線狀態與正在編輯的章節）。

---

### 3.4 章節管理（P0）

**需求描述**：以章節為內容組織單位，支援彈性的結構編輯。

**功能細項**
- 新增章節：在左側目錄樹新增章節。
- **重新命名章節**：雙擊章節標題或右鍵選單即可重新命名。
- 章節排序：拖曳調整章節順序。
- 章節層級：支援「章 → 節」兩層結構（可選配多層巢狀）。
- 刪除章節：刪除前需二次確認；支援軟刪除還原。
- 章節狀態標記：未開始 / 撰寫中 / 待審 / 已完成。

**版面配置（左側目錄樹）**
```
📕 書籍名稱
 ├─ 第一章 緒論              [已完成]
 │   ├─ 1.1 研究背景
 │   └─ 1.2 研究目的
 ├─ 第二章 文獻探討          [撰寫中]
 └─ 第三章 ...               [未開始]
```

**驗收標準**
- 重新命名後即時同步給所有協作者。
- 拖曳排序的結果即時持久化儲存。

---

### 3.5 內容編輯與多人共筆（P0）

**需求描述**：核心編輯區，支援富文本與多人即時協作。

**功能細項**
- 富文本編輯：標題、粗體 / 斜體 / 底線、清單、引用、程式碼區塊、連結、表格。
- **多人即時共筆**：多位作者可同時編輯同一章節，即時看到彼此的游標與輸入內容。
- 衝突處理：採用 CRDT 或 OT（Operational Transformation）演算法處理併發編輯衝突。
- 自動儲存：輸入停頓後 2 秒內自動儲存；顯示「儲存中 / 已儲存」狀態。
- 協作游標：以不同顏色顯示各協作者的游標位置與名稱標籤。
- 離線編輯：暫存於本地，恢復連線後自動同步。

**驗收標準**
- 兩人同時於同段落輸入，內容不丟失、不錯亂。
- 斷網後重連，本地暫存內容正確合併。

---

### 3.6 右側資訊面板 — 統計數據（P0）

**需求描述**：編輯畫面右側固定面板，即時呈現寫作相關數據。

**統計指標**
| 數據 | 範圍 | 說明 |
|------|------|------|
| 目前章節字數 | 章節 | 即時計算（不含空白可選） |
| 全書總字數 | 書籍 | 各章節加總 |
| 段落 / 句子數 | 章節 | 結構統計 |
| 完成進度 | 書籍 | 已完成章節 / 總章節 百分比 |
| 預估閱讀時間 | 章節 / 全書 | 以平均閱讀速度估算 |
| 今日新增字數 | 個人 | 當日寫作量 |
| 各作者貢獻字數 | 書籍 | 協作貢獻佔比 |
| 最近編輯時間 | 章節 | 最後更新時間戳 |

**功能細項**
- 數據隨編輯即時更新。
- 提供「字數目標」設定，顯示進度條與達成率。
- 可切換「章節視角」與「全書視角」。

---

### 3.7 右側資訊面板 — 多媒體管理（P1）

**需求描述**：在右側面板導入並管理多媒體素材，並可插入章節內容。

**支援格式**
| 類型 | 格式 |
|------|------|
| 圖片 | JPG, PNG, GIF, WebP, SVG |
| 影片 | MP4, WebM（或嵌入 YouTube / Vimeo 連結） |
| 音訊 | MP3, WAV |
| 文件 | PDF, DOCX, 一般附件 |

**功能細項**
- 上傳方式：拖曳上傳、檔案選擇、貼上連結。
- 素材庫：以縮圖網格顯示書籍內所有素材，可搜尋、分類。
- 插入內容：將素材一鍵插入目前游標位置。
- 素材中繼資料：檔名、大小、上傳者、引用次數。
- 容量限制：單檔上限與書籍總容量配額（依方案）。

---

### 3.8 版本歷史（P1）
- 自動記錄章節內容快照（時間 / 編輯者）。
- 可預覽歷史版本並比對差異（diff）。
- 可一鍵還原至指定版本。

### 3.9 評論與協作（P2）
- 針對選取段落新增評論。
- @ 提及成員並發送通知。
- 評論狀態：開啟 / 已解決。

## 4. 介面與互動設計

### 4.1 主編輯畫面版面
```
┌──────────────────────────────────────────────────────────────┐
│  頂部列：書籍名稱 │ 線上成員頭像 │ 儲存狀態 │ 分享 │ 設定        │
├────────────┬────────────────────────────────┬─────────────────┤
│            │                                │  右側資訊面板    │
│  左側目錄  │        中央內容編輯區          │ ┌─────────────┐ │
│  （章節樹）│   （富文本 / 多人共筆）        │ │ 📊 統計數據 │ │
│            │                                │ ├─────────────┤ │
│  + 新增章節│                                │ │ 🖼 多媒體   │ │
│            │                                │ └─────────────┘ │
└────────────┴────────────────────────────────┴─────────────────┘
```

### 4.2 互動原則
- 左側目錄、中央編輯、右側面板三欄式佈局，右側面板可收合。
- 所有操作即時回饋，避免「儲存」按鈕等手動動作。
- 響應式設計：桌面優先，平板自動收合側欄。

## 5. 技術規劃（建議）

> 依專案規範，預設採用 Python、JavaScript、HTML；前端以 vanilla JS 為主，除非另行指定框架。

### 5.1 技術堆疊建議
| 層級 | 技術建議 |
|------|----------|
| 前端 | HTML + vanilla JavaScript（編輯器可採用開源富文本核心） |
| 即時協作 | WebSocket + CRDT（如 Yjs）/ OT |
| 後端 | Python（FastAPI 或 Flask） |
| 資料庫 | PostgreSQL（結構資料）+ 物件儲存（多媒體檔案） |
| 快取 / 即時 | Redis（在線狀態、協作 session） |
| 檔案儲存 | 本地 / S3 相容物件儲存 |

### 5.2 資料模型（概念）
```
User（使用者）
 └─ id, name, email, avatar

Book（書籍）
 └─ id, title, description, cover, status, owner_id, created_at

BookMember（成員關聯）
 └─ id, book_id, user_id, role

Chapter（章節）
 └─ id, book_id, parent_id, title, order, status, updated_at

ChapterContent（章節內容）
 └─ id, chapter_id, content(rich/CRDT doc), version

MediaAsset（多媒體素材）
 └─ id, book_id, type, url, filename, size, uploaded_by

ContentVersion（版本快照）
 └─ id, chapter_id, snapshot, editor_id, created_at

Comment（評論）
 └─ id, chapter_id, anchor, content, author_id, status
```

### 5.3 主要 API（概念）
| 方法 | 路徑 | 說明 |
|------|------|------|
| GET | /api/books | 取得書籍列表 |
| POST | /api/books | 建立書籍 |
| PATCH | /api/books/{id} | 更新書籍 |
| POST | /api/books/{id}/members | 邀請成員 |
| POST | /api/books/{id}/chapters | 新增章節 |
| PATCH | /api/chapters/{id} | 重新命名 / 排序 / 狀態 |
| GET | /api/chapters/{id}/content | 取得章節內容 |
| WS | /ws/chapters/{id} | 即時協作通道 |
| GET | /api/books/{id}/stats | 取得統計數據 |
| POST | /api/books/{id}/media | 上傳多媒體 |

## 6. 非功能需求

| 類別 | 需求 |
|------|------|
| 效能 | 章節載入 < 1.5s；協作同步延遲 < 200ms |
| 安全 | 權限驗證、傳輸 HTTPS、檔案掃毒、XSS 防護 |
| 可用性 | 服務可用率 ≥ 99.5% |
| 資料安全 | 內容自動備份、版本可還原 |
| 相容性 | 支援 Chrome / Edge / Safari / Firefox 最新兩版 |
| 國際化 | 介面繁體中文優先，預留多語系架構 |

## 9. 附錄：名詞定義
- **CRDT**：無衝突複製資料型別，用於分散式即時協作。
- **OT**：操作轉換，另一種即時協作衝突解決機制。
- **軟刪除**：標記刪除但保留資料，可還原。
- **P0 / P1 / P2**：功能優先級，P0 最高（必做）。
