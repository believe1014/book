## 1. 設計概述與目標

協作撰書系統是一個以「**書籍 → 章節 → 內容**」三層結構組織的多人線上寫作環境。它要服務兩種心境截然不同的使用者：一邊是需要掌控全局、組裝團隊與結構的 **Owner（主編）**，另一邊是進入深度心流、只想專注打字的 **Editor（共同作者）**。設計核心是讓「結構管理」與「沉浸寫作」在同一個三欄式介面中和諧共存——左側目錄樹負責結構、中央編輯區負責心流、右側面板負責回饋（統計與素材），而所有儲存、同步、協作狀態都以「即時、隱形」的方式回饋，使用者永遠不需要按「儲存」。

### 設計目標（對齊 PRD §2.2 KPI 與 spec 驗收標準）

| # | 設計目標 | 來源 / 衡量 |
|---|----------|------------|
| G1 | 從書架建立書籍到進入編輯畫面 ≤ 3 步、可在 3 秒內完成 | PRD 3.2 驗收、US-01 |
| G2 | 自動儲存對使用者「無感」：停止輸入 2s 內觸發，狀態變化以淡入淡出呈現、不打斷打字 | spec FR-41/43、PRD KPI「自動儲存延遲 < 2s」 |
| G3 | 編輯區為絕對視覺主體：中央內容區佔據最大視覺權重，左右欄皆可收合讓出空間 | PRD §4.2、spec §1.1 |
| G4 | 協作狀態零猜測：任何時刻使用者都能一眼看出「誰在線、誰在編輯哪章、我是否握有編輯權」 | US-05、FR-44/50/51 |
| G5 | 危險操作可回復、可預期：軟刪除、版本還原、覆寫引用素材皆有明確確認與還原路徑 | US-03/US-14、FR-15/72/84 |
| G6 | 全介面達 WCAG 2.1 AA；繁中長文閱讀舒適（行高、字級、護眼底色） | spec §8.4 國際化/安全、PRD §6 |

## 2. 設計原則

1. **寫作優先（Writing-first）**：中央編輯區永遠是視覺主體，最大內容寬度約 720px 置中以利長文閱讀；工具列、章節樹、資訊面板皆為輔助，可收合、退居邊緣，絕不在打字時跳出彈窗或位移版面。任何「是否該打斷使用者」的爭議，一律站在不打斷的一邊。

2. **儲存是系統的責任，不是使用者的責任**：介面中沒有「儲存」按鈕。狀態（`儲存中… / 已儲存 / 連線中斷已暫存`）以頂部列一個低調指示器呈現，靠顏色與淡入淡出而非彈窗傳達。離線時自動轉為本地暫存並明示，回線後自動同步（FR-41、§6.2 離線分支）。

3. **協作可見、衝突可預期**：因 v1.0 採「章節軟鎖定 + 廣播」（spec §1.4），介面必須誠實呈現此取捨——進入他人正在編輯的章節時，明確顯示「唯讀・即時觀看」橫幅與編輯權持有者，而非讓使用者打了字才被 423 拒絕。線上成員、游標、編輯權變更都即時反映。

4. **結構即導覽**：左側章節樹同時是「書的目錄」與「主要導覽」。它要能拖曳重排、雙擊改名、以色點顯示狀態、嚴守最多兩層（第三層拖放即時拒絕並回彈，對應 FR-31/§6.3），讓使用者把它當大綱工具直接思考。

5. **破壞性操作皆可回復**：刪除一律軟刪除並提供 30 天還原入口；還原版本是「新增一個版本」而非覆寫歷史（FR-72）；刪除被引用素材（`ref_count > 0`）需二次確認。介面用語要明確區分「移到回收桶（可還原）」與真正不可逆的動作。

6. **數據即時但安靜**：右側統計面板隨打字即時更新字數/進度，但更新用數字滾動或淡入，不用閃爍或聲響，符合「即時回饋卻不干擾心流」。

## 3. 使用者與情境

角色取自 spec §2.1 與 PRD §3.3 權限矩陣。權限是設計可見性與可用性的硬約束：UI 不應呈現使用者無權執行的主要操作（或以 disabled + 說明呈現）。

| 角色 | 主要目標 | 主要畫面 | 情境與情緒 | 權限重點（PRD §3.3） |
|------|----------|----------|------------|----------------------|
| **Owner（主編/擁有者）** | 開專案、組團隊、掌控結構與全書進度 | 書架、書籍設定、成員管理、編輯畫面（全書視角統計） | 籌備期較理性、需要控制感；關注「進度條」與「各作者貢獻」 | 全部；唯一能刪除書籍、管理成員、調整角色 |
| **Editor（共同作者）** | 專注撰寫指定章節、不與人衝突 | 編輯畫面（章節視角）、媒體面板、版本歷史 | 深度心流；最怕被打斷、最怕內容遺失或被覆蓋 | 章節 CRUD、編輯內容、上傳媒體；不可管理成員/刪書 |
| **Reviewer（審閱者）** | 檢視內容、（未來）評論 | 編輯畫面（唯讀）、版本歷史 | 略過客；需要清楚知道「我不能編輯」 | 僅檢視（評論 P2 延後）；編輯區為唯讀態 |
| **Viewer（讀者）** | 唯讀閱讀 | 編輯畫面（純閱讀，隱藏編輯工具列） | 純讀者；介面應像閱讀器 | 僅檢視內容 |

**新使用者（首次註冊/被邀請）** 為跨角色情境：可能透過邀請 token 連結首次進站（FR-21），落地頁應在註冊/登入後直接帶入被邀請的書籍，而非丟到空書架。

## 4. 資訊架構 (IA)

### 4.1 全域導覽模型

採 **「精簡頂部列 + 情境式側欄」** 模型，而非傳統全域側邊欄：

- **書架層（管理情境）**：頂部列含 Logo／全站搜尋／使用者選單；主體為書籍卡片網格。導覽淺、以內容為主。
- **編輯層（寫作情境）**：進入單一書籍即切換為三欄佈局，頂部列收斂為「書名切換器 + 協作狀態 + 儲存狀態 + 分享/設定」。全站導覽退到書名切換器後面，把螢幕讓給寫作。

理由：本產品 90% 的時間花在單一書籍的編輯畫面，全域導覽若常駐側欄會與「左側章節樹」打架、並侵蝕編輯區。情境式切換讓每個情境只保留該情境需要的導覽。

### 4.2 Sitemap（文字樹）

```
（未登入）
├─ /login            登入
├─ /register         註冊（可帶 ?invite=<token>）
└─ /invite/:token    邀請落地（未登入→導向註冊/登入後自動接受）

（已登入）
└─ /  書架 Bookshelf（預設首頁）
   ├─ 搜尋 / 排序 / 狀態篩選列
   ├─ 書籍卡片網格
   ├─ [建立書籍] 對話框
   └─ /trash        回收桶（軟刪除書籍，30 天內可還原）
   │
   └─ /books/:id  書籍編輯畫面（三欄式，預設落地）
      ├─ 左欄：章節樹（含「＋新增章節」、拖曳排序、右鍵選單）
      ├─ 中欄：富文本編輯區（含/唯讀觀看橫幅）
      ├─ 右欄：資訊面板（分頁切換）
      │   ├─ 統計分頁（章節視角 / 全書視角）
      │   └─ 媒體分頁（素材庫網格 + 上傳）
      ├─ 章節版本歷史（抽屜/對話框，由章節選單開啟）
      └─ 書籍設定（對話框/抽屜）
          ├─ 基本資料（書名/簡介/封面/標籤/狀態/字數目標）
          ├─ 成員管理（列表 + 邀請）
          └─ 危險區（軟刪除書籍）

      └─（P2 預留，本版不實作）段落評論側欄
```

## 5. 使用者流程 (User Flows)

> 標註法：`[畫面名]`、`[[主要動作]]`；分支 `├─ / └─`；追溯 US-xx / FR-xx。

### 流程 1：建立書籍（US-01、FR-10、§6.1）
```
前置：已登入
1. [書架] 使用者點 [[＋ 建立書籍]]
2. [建立書籍對話框] 填 書名(必填,1–200字)、簡介(選填)、標籤(選填) → [[建立]]
   ├─ 書名空白/超長 → 即時欄位錯誤「請輸入 1–200 字書名」，[[建立]] 維持 disabled
   └─ 成功(201) → 後端建 book(status=draft) + 自建 owner 成員(FR-10)
3. [編輯畫面] 自動導向；章節樹為空 → 顯示 empty state，焦點落在 [[＋新增第一章]]
   └─ 建立失敗(500) → Toast「建立失敗，請重試」，停留對話框並保留輸入
```

### 流程 2：開啟書架找書（US-02、FR-11）
```
1. [書架] 載入 → loading 顯示卡片骨架屏
   ├─ 無任何書 → empty state：插畫 + 「還沒有書，建立你的第一本吧」+ [[建立書籍]]
   └─ 有書 → 卡片網格（封面/書名/狀態徽章/字數/進度條/最近編輯）
2. 使用者於搜尋框輸入書名 → debounce 後 GET /books?search= → 即時過濾
3. 切換 排序(最近編輯▾/建立時間/書名) 或 狀態篩選(全部/草稿/進行中/完成/封存)
   └─ 篩選後無結果 → 「找不到符合的書籍」+ [清除篩選]
4. 點卡片 → 導向 /books/:id 編輯畫面
```

### 流程 3：邀請成員（US-04、FR-20/21、§6.4）
```
前置：身份為 Owner
1. [編輯畫面] 頂部 [分享] 或 [設定→成員] → 開 [成員管理]
2. [成員管理] 既有成員列表（頭像/姓名/角色下拉/移除）+ [[＋邀請成員]]
3. [邀請對話框] 輸入 Email + 選角色(editor/reviewer/viewer) → [[送出邀請]]
   ├─ Email 格式錯 → 即時錯誤
   ├─ 邀請自己 → 400「不能邀請自己」
   ├─ 已是成員 → 409「該使用者已是成員」
   ├─ Email 已註冊 → 直接建立成員(或待接受) → 列表即時新增該成員
   └─ Email 未註冊 → 建立 pending 邀請(FR-21) → 列表顯示「⏳ 待接受」列 + [複製邀請連結]
4. （非 Owner 進入設定）→ 成員分頁僅唯讀，邀請按鈕 disabled 並提示「僅擁有者可管理成員」
```

### 流程 4：撰寫內容 + 自動儲存 + 衝突（US-09、FR-41/42/43、§6.2）
```
前置：使用者於章節握有編輯權
1. [編輯區] 使用者輸入富文本
2. 停止輸入 2s(debounce) → 儲存指示器轉「儲存中…」→ PATCH content(base_version)
   ├─ 版本相同(200) → version+1、寫快照、回新 version/word_count
   │     → 指示器轉「✓ 已儲存」，右側統計字數即時更新；1.5s 後淡為灰色靜態
   ├─ 版本衝突(409, base_version 落後) →
   │     指示器轉「⚠ 內容已更新」→ 橫幅:「他人已更新此章節」[檢視最新並合併]
   │     → 自動 GET 最新內容，v1.0 以最後寫入者為準 + 明確提示(§6.2)
   ├─ 非編輯權持有者(423) → 不該發生(UI 已鎖唯讀)；若發生則轉唯讀並提示
   └─ 離線/WS 斷線 → 指示器轉「⚠ 連線中斷・已本地暫存」→ 暫存 localStorage
         → 回線後比對版本再送(§6.2 離線分支)
```

### 流程 5：即時協作與編輯權（US-05、US-10、FR-44/45/50/51）
```
1. [編輯區] 開啟章節 → 建立 WS /ws/chapters/:id?token=
   ├─ 驗證失敗(4401) → 橫幅「連線逾時，請重新整理」
   └─ 無權限(4403) → 導回書架並 Toast「無權限存取」
2. 收 presence → 頂部列線上頭像 + 章節樹該章節旁顯示「✎ 對方姓名 編輯中」
3. 章節無人持有編輯權 → 本使用者自動取得(POST lock)，編輯區可寫
   └─ 章節已被他人持有 → 編輯區頂部黃色橫幅「✎ Amy 正在編輯・唯讀觀看」
        → 即時接收 cursor/content_updated 並自動拉新內容(FR-52)
4. 持有者閒置 60s 或斷線 → lock_changed 廣播釋放 → 唯讀端橫幅轉
     「編輯權已釋放」[接手編輯]
```

### 流程 6：章節拖曳排序與層級（US-07/US-08、FR-33、§6.3）
```
1. [章節樹] 長按/拖動節點 → 出現插入指示線、被拖節點半透明
2. 放開 → 前端組 [{id,parent_id,order_index}] → PATCH reorder（單一交易）
   ├─ 嘗試使「節」變第三層(parent 也有 parent) → 拖放即時拒絕、節點回彈 +
   │     Toast「最多支援兩層結構」(FR-31/§6.3)
   ├─ 成功 → 樂觀更新樹狀；廣播給協作者即時同步(PRD 3.4 驗收)
   └─ 失敗(403/400) → 回退至原順序 + Toast
```

### 流程 7：插入媒體素材（US-13、FR-80/82/84、§6.5）
```
1. [媒體面板] [[＋上傳]]（拖曳/選檔/貼連結）
2. 驗證副檔名/MIME、單檔 ≤50MB、書籍配額 ≤1GB
   ├─ 格式不符(400) → 行內錯誤「不支援的格式」
   ├─ 單檔超限(413) → 「檔案超過 50MB 上限」
   ├─ 配額超限(413) → 「書籍素材已達 1GB，請先清理」
   └─ 成功 → 縮圖出現於網格、顯示上傳進度條→完成
3. 將焦點置於編輯區游標 → 媒體縮圖點 [插入] → 插入游標處，ref_count+1
4. 刪除素材：若 ref_count>0 → 二次確認對話框「此素材已被引用 N 次，仍要刪除？」(FR-84)
```

### 流程 8：版本還原（US-14、FR-71/72、§5.7）
```
1. [章節選單] 點「版本歷史」→ 開 [版本歷史] 抽屜（分頁清單：時間/編輯者/字數）
2. 點某版本 → 右側預覽該版本內容（唯讀）
3. [[還原此版本]] → 確認「將以此版本內容建立一個新版本（不會刪除其他歷史）」
   └─ 確認 → POST .../restore → 新版本成為目前內容、編輯區刷新、Toast「已還原」
```

### 流程 9：刪除與還原書籍（US-03、FR-14/15、§6.6）
```
1. [書籍設定→危險區] [[刪除書籍]]（僅 Owner 可見可用）
2. 確認對話框「移至回收桶，30 天內可還原」 → 確認 → 設 deleted_at → 導回書架
3. [回收桶] 列出已刪書籍 + 剩餘天數徽章
   ├─ [還原] → POST restore → 回到書架
   └─ 逾 30 天 → restore 回 410 → 「已逾還原期限，無法復原」
```

## 6. 畫面清單 (Screen Inventory)

| # | 畫面 | 目的 | 進入點 | 關鍵元件 | 主要操作 | 可見角色 |
|---|------|------|--------|----------|----------|----------|
| S1 | 登入 / 註冊 | 取得 JWT | `/login` `/register`、邀請連結 | 表單、錯誤提示 | 登入/註冊 | 訪客 |
| S2 | 邀請落地 | 接受邀請加入書籍 | `/invite/:token` | 書名/邀請者資訊、接受鈕 | 接受、登入後自動加入 | 訪客/已登入 |
| S3 | 書架 | 列出我參與的書、進入編輯 | `/` 登入後首頁 | 搜尋列、排序/篩選、卡片網格、建立鈕 | 搜尋/排序/篩選/建立/開啟 | 全部 |

## 8. 元件規格與狀態 (Components & States)

> 通用態：default / hover / focus(鍵盤) / active / disabled / loading / error / empty。以下僅列與預設不同、且對本產品有意義的態。

### 元件：書籍卡片 (BookCard)
| 狀態 | 視覺 | 行為 |
|------|------|------|
| default | 封面+書名+狀態徽章+進度條+相對時間 | 整卡可點進入編輯畫面 |
| hover | 輕微抬升（shadow/md）、封面微放大 | 提示可點 |
| loading | 骨架屏（灰塊封面+兩條灰線），避免版面跳動 | 不可點 |
| empty(整頁) | 插畫 + 「還沒有書，建立你的第一本吧」+ [[建立書籍]] | CTA 開 S4 |
| error(整頁) | 「載入書架失敗」+ [重試] | 重新 GET /books |
| 無封面 | 以書名首字 + 主題色塊作為佔位封面 | 同 default |

### 元件：章節樹節點 (ChapterTreeItem)
| 狀態 | 視覺 | 行為 |
|------|------|------|
| default | 標題 + 狀態色點（✓/●/待審/○） | 點擊開啟該章節 |
| hover | 背景淡灰、顯示 ⋮ 選單鈕 | 提示可操作 |
| selected | 左側強調色條 + 粗體 | 目前編輯章節 |
| co-editing | 標題旁「✎ 姓名」徽章 | 顯示他人正在編輯（US-05） |
| editing(rename) | 標題變輸入框、全選文字 | Enter 確認 / Esc 取消（PRD 3.4） |
| dragging | 半透明 + 插入指示線 | 放開即 reorder 持久化 |
| drop-rejected | 插入線轉紅、回彈 | 阻止第三層（FR-31） |
| disabled(無權限) | 灰階、無 ⋮ 選單 | R/V 僅可點擊檢視 |

### 元件：富文本編輯器 (RichTextEditor)
| 狀態 | 視覺 | 行為 |
|------|------|------|
| editable | 工具列可用、游標可見 | 輸入→debounce 2s 自動儲存（FR-41） |
| readonly(角色) | 工具列隱藏、淺灰底 | Reviewer/Viewer 僅閱讀 |
| readonly(鎖定) | 頂部黃色橫幅 + 工具列灰階 | 他人持有編輯權（FR-44，見 7.2b） |
| presence | 他人彩色游標 + 名稱標籤 | 即時顯示（FR-51 cursor） |
| loading | 內容區骨架行 | GET content 中 |
| conflict | 橫幅「他人已更新」+[檢視最新] | 409 衝突（§6.2） |
| offline | 頂部「連線中斷・已本地暫存」 | 暫存 localStorage（§6.2） |
| empty(章節) | 淡色提示「開始撰寫第二章…」 | 點擊聚焦即可輸入 |

### 元件：儲存狀態指示器 (SaveStatusIndicator)
| 狀態 | 視覺 | 行為 |
|------|------|------|
| idle/saved | 灰色「✓ 已儲存」 | 靜態 |
| saving | 主題色「● 儲存中…」 | PATCH 進行中 |
| just-saved | 綠色「✓ 已儲存」 | 成功後 1.5s 內淡為灰（原則 2） |
| conflict | 琥珀「⚠ 內容已更新」 | 可點開衝突橫幅 |
| offline | 紅「⚠ 連線中斷・已暫存」 | 回線自動同步 |

### 元件：統計面板 (StatsPanel)
| 狀態 | 視覺 | 行為 |
|------|------|------|
| default | 字數/段落/閱讀/進度/目標/今日/貢獻者 | 隨打字即時更新（數字滾動，原則 6） |
| view-toggle | 章節視角 ↔ 全書視角 | 切換資料範圍（PRD 3.6） |

## 9. 互動與動效 (Interaction & Motion)

| 互動 | 觸發 → 回饋 → 結果 | 動效/時序 |
|------|---------------------|-----------|
| 自動儲存（FR-41/43） | 停止輸入 2s → 指示器「儲存中…」→ 成功「✓已儲存」→ 1.5s 淡為灰 | 顏色淡入 120ms；淡出 ease-out |
| 字數即時更新（FR-60） | 打字 → 右側數字變化 | 數字短滾動 ~200ms，無閃爍（原則 6） |
| 章節拖曳排序（FR-33） | 拖動 → 插入指示線 + 被拖半透明 → 放開重排 | 節點 transform 150ms ease；非法落點回彈 200ms |
| 章節改名（PRD 3.4） | 雙擊標題 → 就地變輸入框全選 → Enter 確認 | 即時、無模態 |
| 協作游標（FR-51） | 收 cursor → 對方彩色游標 + 名稱標籤平滑移動 | 位置插值 ~100ms 跟手 |
| 在線/編輯權變更（FR-44/45） | 收 presence/lock_changed → 頭像/橫幅淡入更新 | 橫幅滑入 200ms |
| 衝突提示（§6.2） | 409 → 琥珀橫幅滑入 + 指示器轉態 | 滑入 200ms，需使用者操作才消失 |
| 檔案上傳（FR-80） | 拖入 → 拖放區高亮邊框 → 縮圖 + 進度條 → 完成抖動回饋 | 高亮 100ms；進度即時 |
| 插入素材（US-13） | 點 [插入] → 內容游標處出現素材 + 「已插入」micro-toast | 插入淡入 150ms |
| 面板收合（G3） | 點收合 → 右欄滑出、編輯區補滿 | 寬度過渡 200ms ease-in-out |
| Modal 開關 | 開：遮罩淡入 + 內容微縮放入；關：反向 | 150ms；尊重 prefers-reduced-motion |

> 動效原則：短（≤250ms）、目的性（傳達狀態變化而非裝飾）、可關閉（`prefers-reduced-motion` 一律退化為即時切換，§12）。

## 11. 響應式設計 (Responsive)

| 斷點 | 寬度 | 佈局 |
|------|------|------|
| mobile | < 640px | 單欄：編輯區獨佔；章節樹收進左側抽屜（`☰`），資訊面板收進底部分頁 + bottom sheet（見 7.6）。頂部列僅留 ☰/書名/儲存點。 |
| tablet | 640–1024px | 兩欄：章節樹 + 編輯區；右側資訊面板預設收合，以右上 `⟩` 浮出為覆蓋抽屜。 |
| desktop | > 1024px | 完整三欄（章節樹 280 │ 編輯區 flex │ 資訊面板 320），左右皆可手動收合。 |

轉換規則：
- **三欄 → 兩欄**：先收「右側資訊面板」（次要回饋），保留結構 + 內容。
- **兩欄 → 單欄**：再把「章節樹」收進抽屜；編輯區成為唯一主畫面（呼應 G3）。
- 拖曳排序在 mobile 改為長按進入排序模式 + 上下箭頭/拖動把手，降低誤觸。
- 對話框（建立書籍/邀請/設定）在 mobile 改為全螢幕 sheet。

## 12. 無障礙設計 (Accessibility)

對應 spec §8.4（國際化/XSS）與 PRD §6，目標 **WCAG 2.1 AA**。

- **對比**：正文/UI 文字對背景 ≥ 4.5:1，大字/圖示 ≥ 3:1（見 §10 對比註記）。狀態色點一律「色 + 文字/圖示」雙重編碼，不單靠顏色傳達（色盲友善）。
- **鍵盤操作**：所有互動可純鍵盤完成。Tab 焦點順序：頂部列 → 章節樹 → 編輯區 → 資訊面板。焦點環清晰（2px brand 色 outline）。快捷鍵建議：`Ctrl/Cmd+S` 立即儲存、`Ctrl/Cmd+B/I/U` 文字樣式、`F2`/雙擊 改名章節、`Esc` 關閉浮層。
- **章節樹（custom tree）**：`role="tree"`、節點 `role="treeitem"`、`aria-expanded`、`aria-level`（限 1–2）、`aria-selected`；上下鍵移動、左右鍵收合/展開、Enter 開啟、F2 改名。
- **Modal**：`role="dialog"` + `aria-modal="true"` + `aria-labelledby`；開啟 focus trap、關閉後焦點返回觸發元素。
- **Toast/狀態**：儲存狀態與錯誤用 `aria-live="polite"`（衝突/錯誤用 `assertive`）播報，讓螢幕報讀者得知「已儲存/連線中斷」。
- **協作游標**：純視覺資訊需提供文字替代——線上成員以可讀清單呈現（`aria-label="Amy 正在編輯 2.1 國內研究"`）。
- **編輯器**：富文本工具列按鈕具 `aria-pressed`；唯讀態以 `aria-readonly="true"` 標示並由橫幅文字說明原因。
- **動效**：尊重 `prefers-reduced-motion`，所有過渡退化為即時切換（§9）。
- **表單**：欄位 `label` 關聯、錯誤以 `aria-describedby` 連結錯誤訊息、`aria-invalid`。
- **安全相關 UX**：富文本渲染需消毒（XSS，spec §8.4）——插入連結/HTML 時於 UI 層即提示不安全內容被移除。

## 13. 待確認事項與假設 (Open Questions & Assumptions)

> 以下為 spec/PRD 未明確規範、本設計做出的決定，請確認或修正。

1. **平台/技術假設**：spec §8.1 採 React 18 + Vite（PRD §5.1 曾提 vanilla JS）。本文件以 **React 元件化 + 三欄 SPA** 為前提。若最終採 vanilla JS，元件規格仍適用、但需自行實作狀態管理。→ 確認前端框架。
2. **唯讀觀看的呈現程度**：v1.0 為「章節軟鎖定」（spec §1.4）。本設計選擇「進入即顯示唯讀橫幅 + 即時觀看」而非「允許輸入後才 423 拒絕」。是否接受此較保守但較不挫折的做法？
3. **編輯權取得時機**：假設「開啟章節且無人持有時自動取得 lock」。是否要改為「點擊『開始編輯』才取得」，以避免只想瀏覽的人佔用編輯權？
4. **書架預設視圖**：spec/PRD 提卡片或列表二擇。本設計預設**卡片網格**（封面導向）。是否需要列表/網格切換？
5. **封面與標籤 UI**：spec 有 cover_url/tags 欄位但無上傳/管理流程細節。本設計假設封面於「書籍設定」上傳、無封面以首字色塊佔位；標籤為自由輸入膠囊。→ 確認標籤是否需預設分類。
6. **邀請寄信**：spec §8.3 明示 v1.0 不含實際寄信，以 token 連結為主。故 UI 以「複製邀請連結」為主要交付方式，未設計收件匣/通知中心。→ 確認是否接受。
7. **衝突解決 UX**：v1.0「最後寫入者為準 + 提示」（§6.2）。本設計提供「檢視最新並合併（手動）」橫幅，但不做自動三方合併。是否足夠，或需 diff 對照？
8. **版本 diff**：PRD 3.8 提「比對差異 (diff)」，spec §5.7 僅有列出/取得/還原。本設計版本歷史為「清單 + 唯讀預覽 + 還原」，**未含 diff 視圖**（標記為對齊 spec）。是否要補 diff？
9. **章節狀態誰可設定**：spec 有四狀態但未明定權限。假設 Owner/Editor 可改章節狀態，Reviewer/Viewer 不可。→ 確認。
10. **「今日新增字數」維度**：spec FR-63 為個人維度（依當日版本快照差值）。UI 顯示為「今日 +N 字（你）」。是否需團隊維度？
11. **回收桶位置**：假設置於書架選單。是否需獨立頁面/導覽入口？
12. **P2 評論**：IA 已預留段落評論側欄位置，但本版不設計樣式與互動（對齊 spec 延後）。
